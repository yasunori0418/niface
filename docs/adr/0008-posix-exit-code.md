# ADR-0008: exit code は POSIX 慣行のまま status と連動させる

- ステータス: 採用
- 日付: 2026-07-05
- 関連: `spec/v1/spec.md`（§1）, ADR-0002

## 背景

ツールの exit code に独自の意味体系（コードごとに固有の失敗種別を割り当てる等）を作るか、POSIX 慣行に従うかを決める必要がある。

## 決定

- exit code は POSIX 慣行のまま（0 = 成功、非 0 = 失敗）とする。
- 最上位 `status` と連動させる（`success` ⇔ 0、`error` ⇔ 非 0）。消費側が依存してよいのはこの対応のみ。
- 独自の exit code 体系は作らない。

## 根拠

- ncompose がツールをシェルコマンドとして実行する以上、exit code の意味論はシェル文化に馴染むものが最良。
- nput の 0 / 1 / 2 も POSIX 準拠の範囲内で 1 / 2 の役割を明示定義しているだけであり、消費側契約は「0 か非 0 か」に限定する。個別コードの意味はツールの裁量に委ね、規格は連動だけを定める。

## 影響

- `spec/v1/spec.md` §1。
- 各ツールの実装（exit code と `status` の連動）。

## 棄却した代替案

- **独自の exit code 体系**: シェル文化と乖離し、消費側に学習コストを課す。エラー種別は `code`（→ ADR-0005）で機械識別できるので exit code に持たせる必要がない。
- **exit code を `status` と独立に決める**: 「0 なのに status error」のような不整合が生まれ、`&&` 連結の前提が崩れる。
